# Installation Guide

## Quick Start

### Option 1: npx (Recommended)

Install and configure Light-Mem with a single command:

```bash
npx light-mem install
```

The interactive installer will:
- Run a runtime check (auto-installs Bun and uv if missing)
- Detect Claude Code and offer to install it if not found
- Prompt for the Claude model used to compress observations (Haiku / Sonnet / Opus)
- Copy plugin files into the marketplace directory and register the plugin
- Auto-start the worker service

### Option 2: Plugin Marketplace

Install Light-Mem directly from the plugin marketplace inside Claude Code:

```bash
/plugin marketplace add light-mem/light-mem
/plugin install light-mem
```

Both methods will automatically configure hooks and start the worker service. Start a new Claude Code session and you'll see context from previous sessions automatically loaded.

> **Important:** Light-Mem is published on npm, but running `npm install -g light-mem` installs the
> **SDK/library only**. It does **not** register plugin hooks or start the worker service.
> Always install via `npx light-mem install` or the `/plugin` commands above.

## System Requirements

- **Node.js**: 20.0.0 or higher
- **Bun** ≥ 1.0 (auto-installed by `npx light-mem install` if missing)
- **uv** (auto-installed if missing — provides Python for Chroma's embedding service)
- **Claude Code**
- **SQLite 3**: bundled via `bun:sqlite`

## Advanced Installation

For development or testing, you can clone and build from source:

### Clone and Build

```bash
# Clone the repository
git clone https://github.com/Ronald-Estacion_NordTech/light-mem.git
cd light-mem

# Install dependencies
npm install

# Build hooks and worker service
npm run build

# Worker service will auto-start on first Claude Code session
# Or manually start with:
npm run worker:start

# Verify worker is running
npm run worker:status
```

### Post-Installation Verification

#### 1. Automatic Dependency Installation

Dependencies are installed automatically by `npx light-mem install` and `npx light-mem repair`. Heavy lifting (Bun + uv install, `bun install` inside the plugin cache) happens behind a visible installer spinner. The Setup hook only performs a sub-100ms `version-check.js` read of the `.install-version` marker — on mismatch it prints `run: npx light-mem repair` to stderr and exits 0, so it never blocks a session. Works cross-platform on Windows, macOS, and Linux.

#### 2. Verify Plugin Installation

Check that hooks are configured in Claude Code:
```bash
cat plugin/hooks/hooks.json
```

#### 3. Data Directory Location

Data is stored in `~/.light-mem/`:
- Database: `~/.light-mem/light-mem.db`
- PID file: `~/.light-mem/.worker.pid`
- Port file: `~/.light-mem/.worker.port`
- Logs: `~/.light-mem/logs/worker-YYYY-MM-DD.log`
- Settings: `~/.light-mem/settings.json`

Override with environment variable:
```bash
export LIGHT_MEM_DATA_DIR=/custom/path
```

#### 4. Check Worker Logs

```bash
npm run worker:logs
```

#### 5. Test Context Retrieval

```bash
npm run test:context
```

## Upgrading

Upgrades are automatic when updating via the plugin marketplace. After an external upgrade (for example `claude plugin update`), the Setup hook detects a version-marker mismatch and asks you to run `npx light-mem repair`, which installs any missing runtime dependencies and refreshes the marker.

See [CHANGELOG](https://github.com/Ronald-Estacion_NordTech/light-mem/blob/main/CHANGELOG.md) for complete version history.
